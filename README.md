# tg_bot

Минимальный Telegram-бот (слот-механика) на `aiogram`.

## Возможности

- Команды:
  - `/start` — приветствие и подсказка по командам.
  - `/balance` — текущий баланс пользователя.
  - `/spin <amount>` — спин со ставкой и расчетом выплаты.
- Валидация ставок:
  - только целое число;
  - между `MIN_BET` и `MAX_BET`;
  - не больше текущего баланса.
- Централизованный логгер.
- Поддержка запуска через **polling** (по умолчанию) и **webhook**.

## Структура

```text
bot/
  main.py

  logger.py
  handlers/
    commands.py
  services/
    game.py
    validation.py
    wallet.py
```

## Локальный запуск (polling)

1. Создайте и активируйте виртуальное окружение.
2. Установите зависимости:

```bash
pip install -r requirements.txt
```

3. Подготовьте `.env` на основе примера:

```bash
cp .env.example .env
```

4. Заполните минимум:

- `BOT_TOKEN`
- `DB_URL`
- `MIN_BET`
- `MAX_BET`

5. Запустите:

```bash
python -m bot.main
```

## Работа в группах

Перед добавлением бота в групповые чаты настройте его в **BotFather**:

1. Включите `Allow Groups` через команду `/setjoingroups`.
2. Настройте privacy mode через `/setprivacy`:
   - **ON** — бот видит только команды, упоминания и сообщения-ответы (`reply`) на его сообщения.
   - **OFF** — бот видит больше сообщений в чате (полезно, если это понадобится для будущей логики).

В группе команды можно вызывать в двух форматах:

- `/spin`
- `/spin@<bot_username>`

Чек-лист прав бота в группе:

- [ ] Бот добавлен в группу.
- [ ] Боту разрешено отправлять сообщения (минимально необходимое право).
- [ ] (Опционально) Боту разрешено удалять/редактировать сообщения и закреплять их, если такая логика будет добавлена позже.

Пример сценария в одном групповом чате:

1. Пользователь **Alice** отправляет `/spin 100`.
2. Пользователь **Bob** отправляет `/spin 50`.
3. Бот обрабатывает ставки отдельно для каждого пользователя.
4. Баланс **Alice** меняется только у **Alice**, баланс **Bob** — только у **Bob**.

## Прод-запуск (webhook)

Для webhook установите переменные:

- `WEBHOOK_URL` — публичный HTTPS URL, например `https://bot.example.com`
- `WEBHOOK_PATH` — путь webhook, например `/webhook`
- `HOST` и `PORT` — адрес и порт локального HTTP-сервера

Пример запуска:

```bash
WEBHOOK_URL=https://bot.example.com HOST=0.0.0.0 PORT=8080 python -m bot.main
```

Важно: сервер должен быть доступен из интернета по HTTPS.
